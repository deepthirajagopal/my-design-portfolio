Case Study 01
Ready1 — Incident Response Orchestration
Agentic SOC platform · End-to-end 0→1 design lead
Agentic UX High Stakes
Enterprise B2B Public
Problem
🔥
Security teams were drowning in fragmented tooling during active incidents
SOC teams managing active security incidents were forced to context-switch across 6–10 disconnected tools — ticketing systems, Slack threads, runbooks, severity trackers — while the clock was running. Critical information lived in silos. Team coordination happened ad-hoc. There was no single system of record for an active incident from detection through post-incident review.
  • Mean time to contain a critical incident: 4–6 hours (industry average 2024)
  • Analyst cognitive load during bridge calls was cited as the #1 source of errors
  • No structured handoff between shifts during multi-day incidents
Research
🔬
6 weeks of embedded research with active SOC teams
  • Contextual inquiry — observed 3 active incident responses alongside Tier 2/3 analysts to map the real task flow (not the idealized one)
  • Job shadowing — followed Incident Response Leads during post-incident reviews to understand which artifacts mattered and which were ceremonial
  • Workflow archaeology — analyzed 40+ historical incident Slack threads to extract ad-hoc patterns that informed the Scribe and Bridge features
  • Persona definition — three core personas: Incident Commander, Analyst, Executive Observer. Each has radically different information needs at T+0, T+1h, T+24h
Key insight: The most experienced responders had developed informal "mental state machines" for tracking incident phase. The design opportunity was making this implicit knowledge explicit and shared across the team.
Process
⚙️
Defining the interaction model for agentic + human collaboration
  • Stage framework design — designed the 6-phase incident lifecycle model (Analysis → Detection → Containment → Eradication → Recovery → Post-Incident) as both a mental model and a primary navigation surface
  • Information hierarchy — ran a card sort with 12 analysts to establish what metadata belongs at each view level (summary card vs. drill-down vs. bridge call)
  • Agentic Scribe design — designed the AI-assisted activity log that captures structured + unstructured updates, reducing the manual burden of incident documentation by 80%
  • Impact surface — co-designed the Impact widget taxonomy (Departments, Users, Accounts, Endpoints, Locations, Servers) with the engineering and data teams
  • Leadership — ran weekly design reviews with CPO and CTO, managed design system contributions across 3 squads, mentored 2 junior designers
Solution
✅
A unified incident orchestration platform with agentic assistance built in
Ready1 gives incident commanders a single pane of glass across the full incident lifecycle. The Stage Indicator surfaces phase progress and task completion across every workstream simultaneously. The Scribe auto-logs all team activity, generating the post-incident report automatically. The Bridge module coordinates standup scheduling, team roster, and live status — eliminating the "who's on the bridge?" Slack thread.

Case Study 02
Forest Druid — Attack Path Visualization
Graph-based threat intelligence UI · Attack surface reduction
Data Viz Security
Complex Systems Public
Problem
🔥
Attack paths through an enterprise's identity graph were invisible to defenders
Active Directory environments contain thousands of interconnected objects — users, groups, OUs, GPOs, service accounts — and attackers exploit the relationships between them (Kerberoasting, ACL abuse, shadow admin paths). Defenders had no visual tool that made these multi-hop attack paths legible without deep graph-theory expertise. Risk lived in the white space between systems.
  • Average enterprise AD environment: 50k–500k objects
  • Security teams could not identify "Tier Zero" assets without running bespoke scripts
  • Existing tools produced adjacency matrices, not human-readable risk narratives
Research
🔬
Learning graph cognition from red teamers and identity architects
  • Expert interviews — 8 in-depth sessions with red teamers and AD architects to understand how they mentally model attack paths vs. how defenders model them (radically different frames)
  • Cognitive task analysis — mapped the "path from domain user to domain admin" mental simulation that expert attackers run, then designed the visualization to support that same simulation for defenders
  • Prior art review — analyzed BloodHound, Maltego, and network topology tools to identify UX failure modes in force-directed graph layouts
  • Key tension — legibility vs. completeness. Showing all paths overwhelms; filtering too aggressively hides real risk
Process
⚙️
Designing a risk-ranked path explorer with progressive disclosure
  • Tiered asset model — designed the "Tier Zero / Tier One / Tier Two" visual hierarchy, giving each asset category a distinct visual weight and interaction affordance
  • Path ranking algorithm collaboration — worked with the data science team to co-design the risk scoring model so that the highest-severity paths surfaced above the fold
  • Contextual drill-down — designed the node detail panel that translates raw graph edges (ACL relationships, delegation chains) into plain-English risk descriptions
  • Remediation flow — designed the "Fix this path" workflow that generates a prioritized remediation checklist directly from the selected attack path
Impact
✅
Attack surface reduction made visible and actionable
The visualization made it possible for identity security teams — without prior graph theory experience — to identify and remediate critical attack paths within a single session, rather than requiring days of manual investigation.
